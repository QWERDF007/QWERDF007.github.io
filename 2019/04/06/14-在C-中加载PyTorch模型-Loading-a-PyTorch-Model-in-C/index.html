Loading a PyTorch Model in C++

正如其名，PyTorch 的主要接口是 Python 编程语言。虽然 Python 在许多需要动态性和易于迭代的场景是合适和首选的语言，但也有很多情况下 Python 的这些特性恰恰是不利的。后者经常应用的一个环境是生产环境，延迟低且部署要求严格。对于生产环境，C++ 通常是首选语言，即使只是将其绑定至另一种语言，如 Java，Rust 或者 Go。接下来的段落将会概括 PyTorch 提供的从已存在的 Python 模型到序列化表示的路径，该序列化表示能从 C++ 加载和执行，不需要依赖 Python。

Step 1: Converting Your PyTorch Model to Torch Script

一个 PyTorch 模型从 Python 至 C++ 的过程被 Torch 脚本实现，PyTorch 模型的表现能被 Torch 脚本编译器理解，编译和序列化。如果从由 “eager” API 编写的现有的 PyTorch 模型开始，首先必须将你的模型转换至 Torch 脚本。在大多数情况下，如下面所讨论的，只需要很少的工作。如果已经有了 Torch 脚本模块，则可以跳过教程的下一节。

将 PyTorch 模型转换为 Torch 脚本存在两种方式。第一种被称为 tracing (追踪)，一种机制，通过使用示例输入评估一次模型，并记录这些输入在模型中的流动，以捕获模型的结构。这适合使用有限的控制流的模型。第二种方法是添加显示的注释至模型中，通知 Torch 脚本编译器它可以直接地解析和编译模型代码，这受限于 Torch 脚本语言的约束。

TIP:
你可以在官方的 Torch Script reference 中找到上述方法的完整的文档，及使用哪种方法的进一步指导。

Step 5: Getting Help and Exploring the API

本教程希望你对 PyTorch 模型从 Python 至 C++ 的路径有一个很好的理解。有了本教程中描述的概念，你应该能够从普通的 “eager” PyTorch 模型出发，到 Python 编译的 ScriptModule，到磁盘上的序列化文件，再到 C++ 中可执行的 script::Module。
