Finetuning Torchvision Models

了解如何对 torchvision 的模型进行微调和特征提取，这些模型都已经在1000类 Imagenet 数据集上预训练过。深入了解如何运用几个现代的 CNN 结构，并将建立微调任意 PyTorch 模型的直觉。由于每个模型结构不同，因此没有可以在所有场景都能工作的微调模板代码。相反，必须查看现有的体系结构，并调整每个模型。

微调，由预训练的模型开始，并为新任务更新模型的全部参数；特征提取，由预训练的模型开始，仅更新最后的层的权重，由该层得到预测值。它之所以叫特征提取，是因为使用预训练的 CNN 作为固定的特征提取器，并且仅修改输出层。

两种迁移学习的方法都遵循下面的步骤：

  • 初始化预训练的模型
  • 改变最后一层的大小来获取与新数据集类别数量相同的输出
  • 定义优化算法，在训练过程更新相应的参数
  • 运行训练步骤

Model Training and Validation Code

函数 train_model 处理给定模型的训练和验证。它接受的参数为 PyTorch 模型，包含数据加载器的字典，损失函数，优化器，训练轮数，标志 inception 模型的布尔标志。该标志使用 Inception v3 模型，因为该结构使用一个辅助输出，并且整个模型的损失同时考虑辅助输出和最终的输出。该函数训练指定轮，并且每轮训练后都跑整个验证。它还跟踪表现最优的模型(根据验证集准确率)，训练结束后返回最优的模型。每轮训练完，都打印训练和验证准确率。

Set Model Parameters’ .requires_grad attribute

函数 set_paramter_requires_grad 当需要特征提取时，将模型里参数的属性 .requires_grad 设为 False。默认情况，如果从头开始训练或者微调，加载预训练的模型的参数都带有 .requires_grad=True。然而，如果进行特征提取，并且仅想改变新的层，则其他层参数不需要梯度。

Initialize and Reshape the Networks

接下来更改模型的网络形状。这不是自动化过程，并且对每个模型都是唯一的。回想一下，CNN 模型的最后一层，通常是全连接层，具有与数据集的类别相同的数目的节点。因为所有的模型都是在 Imagenet 上预训练的，所以它们全都有1000大小的输出层，一个节点对应一个类别。更改最后一层的目的是，使其与之前输入大小一样，并且输出大小和数据集的类别一样。下面将讨论如何单独地更改每个模型的结构。

当进行特征提取，仅需要更改最后一层的参数，或者换种说法，仅需要更新更改过形状的层的参数。因此，不需要计算没有更改的参数的梯度，所以设置 .requires_grad=False。而微调，则所有参数的 .requires_grad 保持默认 True。

Load Data

知道了输入大小，就可以初始化数据变换，图像数据集及数据加载器。模型用硬编码的标准化值预训练的。

Create the Optimizer

创建一个仅更新指定参数的优化器。在加载预训练模型后，但在更改之前，如果 feature_extract=True，则手动设置所有参数的 .requires_grad 属性为 False。然后重新初始化的参数默认带有 .requires_grad=True 。因此所有带有 .requires_grad=True 的参数应该被更新。接着，将这些参数生成一个列表，并将其传递给 SGD 算法构造器。

当进行微调时，列表应该长，并且包含模型所有的参数。然而，当进行特征提取时，列表应该短，且仅包含更改过的层的权重和偏置。为验证这点，打印要学习的参数。

Run Training and Validation Step

最后的步骤是创建模型的损失函数，然后运行指定轮训练和验证函数。取决于指定的轮次，可能在 CPU 上要花费些时间。另外，默认的学习率并不是对所有的模型都最优，所以为取得最大的准确率需要对每个模型单独地微调。
