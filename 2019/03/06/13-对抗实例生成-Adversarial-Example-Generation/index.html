Adversarial Example Generation

研究正在不断地推动 ML 模型更快，更准和更有效。然而，设计和训练模型时经常会忽略模型的安全性和鲁棒性，特别在面对想欺骗模型的对手时。下面将提高关于 ML 模型的安全漏洞的认知，并深入了解对抗机器学习这一热门话题。在图像中添加细微的扰动就可能导致模型性能的巨大差异。通过一个图像分类器上的例子来探索该主题。使用第一个也是最流行的攻击方法之一，快速梯度符号攻击(FGSM)，来欺骗 MNIST 分类器。

Threat Model

对于上下文，有多种分类的对抗攻击，每种都有不同的目标和对攻击者知识的假设。无论如何，首要目标是添加最少的扰动至输入数据，得到错误分类。有几种关于攻击者知识的假设，其中两种是：白盒和黑盒。白盒攻击假设攻击者有模型全部的了解和访问权，包括结构，输入，输出和权重。黑盒攻击假设攻击者仅能访问模型的输入和输出，并不知道潜在的结构或权重。同样还有几种类型的目标，包括错误分类和源/目标错误分类。错误分类意味着对抗者仅想输出分类是错误的，而不关心新的分类是什么。源/目标错误分类意味着对抗者想改变原来属于特定分类的图像，以便分类其为特定的目标。FGSM 攻击是以错误分类为目标的白盒攻击。

Fast Gradient Sign Attack

目前为止，最早也是最流行的对抗攻击被称为快速梯度符号攻击(FGSM)，由 Goodfellow 提出，论文在这Explaining and Harnessing Adversarial Examples。这种攻击非常强大，并直观。它被设计于攻击神经网络，通过借助它们学习的方式—梯度。想法很简单，比起根据反向传播回来的梯度调整权重来最小化损失，这种攻击根据相同的反向传播回来的梯度调整输入来最大化损失。换言之，这种攻击使用了输入数据相关的梯度，然后调整输入来最大化损失。

代码之前，先看看论文中的例子，$x$ 是被正确分类为熊猫的原始输入图像，$y$ 是 $x$ 的真实标签，$ \theta $ 代表模型的参数，$ J(\theta, x, y) $ 是训练网络的损失。该攻击将梯度反向传播回输入数据，来计算 $ \triangledown_x J(\theta, x, y) $。然后，它在该方向上细微调整输入数据，最大化损失。结果干扰了图像，$x’$ 还明显是熊猫时被目标网络错误分类为长臂猿。

先导入需要使用的包

Implementation

本节将讨论输入参数，定义受攻击的模型，然后编写攻击和运行一些测试。

Inputs

仅有三个输入，定义如下：

  • epsilons：运行时使用的 $ \epsilon $ 列表。列表中保留0是很重要的，因为它代表模型在原始测试集上的性能。直观上，epsilon 越大，扰动越明显，但在降低模型准确性方面越有效。由于数据范围为[0,1]，因此 epsilon 值不应该超过1。
  • pretrained_model：预训练的 MNIST 模型的路径。
  • use_cuda：布尔标志，是否使用 CUDA。本例中 GPU 不是必须的。

Model Under Attack

可以训练并保存自己的 MNIST 模型，或者可以下载和使用提供的模型。这儿的网络定义和测试 dataloader 是从 MNIST 示例中复制的。

FGSM Attack

定义函数，通过干扰原始输入创建对抗示例。fgsm_attack 函数接收3输入，image 是原始干净的图像，epsilon 是像素级扰动量 $ \epsilon $，data_grad 是输入图像相关损失的梯度($ \triangledown_x J(\theta, x, y)$)。该函数创建干扰图像：

$perturbed_image = image + epsilon sign(data_grad) = x + \epsilon sign(\triangledown_x J(\theta, x, y)$

最后，为了保持原始的数据范围，干扰后的图像被裁剪至范围[0,1]。

Testing Function

每次调用测试函数都在 MNIST 测试集上执行完整的测试，并报告最终准确率。然而，该函数也接收 epsilon 输入。这是因为 test 函数报告受 $\epsilon$ 强度攻击的模型的准确性。对于每个测试样例，该函数都计算和输入数据相关损失的梯度(data_grad)，通过 fgsm_attack 函数创建干扰图像(perturbed_data)，然后检查干扰例子是否是对抗性的。除了测试模型的准确率，该函数还保存和返回一些成功的对抗示例，以便后续查看。

Run Attack

对于 epsilons 列表中每个值都运行完整的测试，保存每个值的最终准确率和一些成功的对抗示例，并在下一节进行绘制。准确率会随着 $\epsilon$ 增加而降低，$\epsilon = 0$ 将打印无攻击的测试准确率。

Result

Accuracy vs Epsilon

第一个结果是准确率 - $\epsilon$ 图，如之前所说，随着 $\epsilon$ 增加，测试准确率会降低。这是因为更大的 $\epsilon$ 意味着沿着最大化损失方向的步幅越大。注意尽管 $\epsilon$ 是线性增长，但该曲线的趋势不是线性的。

Sample Adversarial Examples

虽然随着 $\epsilon$ 增加，测试准确率下降，但干扰变得更加容易察觉。实际上，攻击者必须考虑准确性下降和可感知性之间的平衡。下面显示一些每个 $\epsilon$ 值对应的成功的对抗示例，每行显示同一 $\epsilon$ 的不同示例，然而，全部示例不管是否添加噪声，人类都可清晰分辨(信了你的邪，都挤成一团了)。
