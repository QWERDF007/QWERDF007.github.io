Neural Transfer using PyTorch

神经风格迁移—Neural Style Transfer

Introduction

Neural-Style 或 Neural-Transfer，允许用新的风格重新生成一张图像。该算法取三张图像，输入图像，内容图像，风格图像，并将改变输入图像使其内容类似内容图像，风格类似风格图像。

Underlying Principle

底层原理很简单：定义两个距离，一个是内容距离(D~C~) ，还有一个是风格距离(D~C~)。D~C~ 测量两图之间的内容有多不同，而 D~S~ 测量两图之间风格有多不同。然后，取输入图像并且转换它，来最小化它与内容图像的内容距离和它与风格图像的风格距离。先加载需要的包，然后开始进行神经风格迁移。

Importing Packages and Selecting a Device

下面是实现神经风格迁移需要用到的包的列表：

  • torch，torch.nn，numpy (不可或缺的)
  • torch.optim (有效的梯度下降)
  • PIL，PIL.Image，matplotlib.pyplot (加载和显示图像)
  • torchvision.transforms (变换 PIL 图像 为 Tensors)
  • torchvision.models (训练或者加载预训练模型)
  • copy (深度拷贝模型，标准库)

接下来选择运行网络的设备，并导入内容和风格图像。在大型图像上运行神经风格迁移算法更久，在 GPU 上运行更快。可以用 torch.cuda.is_available() 来检测是否有 GPU 可用。设置 torch.device 来选择设备，并且 .to(device) 方法可以把 Tensors 或 模块移至想要的设备上。

Loading the Images

导入内容和风格图像。原始的 PIL 图像像素值在0-255之间，但当转换为 torch Tensors 后，它们的像素值被转为0-1之间。图像需要更改大小使得维度相同。torch 库的神经网络是用0-1范围的 Tensors 训练的。如果尝试喂给网络值为0-255的 Tensors，则已经激活的特征图将无法感知预期的内容和风格。然而，Caffe 库的预训练网络是用值为0-255的 Tensors 训练的。

创建一个函数来显示图像，转换它的副本为PIL格式，并且用 plt.imshow 来显示。

Loss Function

Content Loss

内容损失是一个表示一层的内容距离的加权版本的方法。该方法使用网络中处理输入的层的特征图 F~XL~ ，并返回加权后的图像 X 和内容图像 C 的内容距离 w~CL~ ^.^ $D^L_C(X, C)​$。该方法必须知道内容图像的特征图(F~CL~)，来计算内容距离。将该方法实现为一个 torch 模块，接收 F~CL~ 作为输入。 $||F_{XL} - F_{CL}||^2​$ 是两种特征图的均方差，可用 nn.MSELoss 计算。

将内容损失模块直接添加至卷积层后，用来计算内容距离。这样每次输入图像至网络，内容损失会在想要的层上计算，并且由于自动梯度，所有梯度都会被计算。为使内容损失层透明化，定义一个 forward 方法来计算内容损失，同时返回该层的输入。计算出的损失作为参数保存至该模块。

注意：虽然该模块被称为 CententLoss，但它不是真的 PyTorch 损失函数。如果想作为 PyTorch 损失函数来定义内容损失，则必须创建一个 PyTorch 自动求导函数来手动地在 backward 方法内重新计算/实现梯度。

Style Loss

风格损失模块实现类似于内容损失模块。作为网络中的透明层，计算该层的风格损失。为了计算风格损失，需要计算格拉姆矩阵 G~XL~ 。格拉姆矩阵是给定矩阵乘其转置矩阵的结果。在这应用中，给定的矩阵是 L 层的特征图 F~XL~ 的重塑版本。F~XL~ 被重塑为 $\hat{F}{XL}$ ，一个 KxN 矩阵，K 是层 L 的特征图数目，而 N 是任意向量化特征图 $\hat{F}{XL}$ 的长度。例如，$\hat{F}{XL}$ 的第一行对应第一个向量化特征图 $F^1{XL}$ 。

最后，格拉姆矩阵每个元素必须除以矩阵元素的总数来归一化。归一化是为了抵消 $\hat{F}_{XL}$ 具有很大维度 N 而产生很大的值。在梯度下降时，这些很大的值会导致第一层(池化层之前)产生较大的影响。风格特征趋向于在网络中更深的层，因此该归一化步骤是重要的。

风格损失用 G~XL~ 和 G~SL~ 间的均方差计算。

Importing the Model

导入一个预训练的神经网络，将使用论文中的19层的 VGG 网络。PyTorch 的 VGG 实现被分为两个子 Sequential 模块：features (包含卷积层和池化层)，和 classifier (包含全连接层)。需要使用 features 模块来输出各层卷积层，测量内容和风格损失。有些层的行为在训练和评估期间不一样，所以必须使用 .eval() 设置网络为评估模式。

另外，VGG 网络在每个通道均值为 [0.485, 0.456, 0.406] 和方差为 [0.229, 0.224, 0.225] 的图像上训练。因此在图像被送入网络前需要使用它门归一化。

Sequential 模块包含一个子模块的有序列表。例如，vgg19.features 包含一个按正确深度顺序对齐的序列(Conv2d, ReLU, MaxPool2d, Conv2d, ReLU…)。需要在卷积层之后添加内容损失和风格损失。因此，创建一个新的 Sequential 模块，正确插入内容和风格损失。

接下来，选择输入图像，可以使用内容图像的副本或者白噪声。

Gradient Descent

使用 L-BFGS 算法来进行梯度下降。训练输入图像来最小化内容/风格损失。创建一个 PyTorch L-BFGS 优化器 optim.LBFGS，然后将传递图像作为 Tensor 去优化。

最后，必须定义一个函数来执行神经风格迁移。对于网络的每次迭代，都将更新过的输入传入其中，并计算新的损失。将运行每个损失模块的 backward 方法来动态计算它们的梯度。优化器需要一个 “closure” 函数，重新评估模块和返回损失。还有一个限制要解决，该网路可能会尝试优化超过[0,1]的输入图像，可以在每次网络运行时将输入的范围修正为[0,1]。
