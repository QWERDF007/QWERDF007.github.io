然后巴拉巴拉一顿安装，安装完可能会出现黑屏或者无限登录的情况，网上有多种解决方案：

  1. 安装的时候加参数

    • -no-x-check：安装时关闭 X
    • -no-nouveau-check：安装时禁用 Nouveau
    • -no-opengl-files：只安装驱动，不安装 OpenGL

  2. 安装驱动后修改grub

    打开文件 /etch/default/grub

    然后找到 GRUB_CMDLINE_LINUX_DEFAULT="quiet splash"，如果发现有 nomodeset，将其修改为 acpi_osi=linux。

    • nomodeset：在 X 之前不要加载显卡驱动，调用集显驱动
    • acpi_osi=linux：告诉 BIOS 以 Linux 系统启动，调用 Linux 的驱动，因此可以调用 NVIDIA 驱动

    http://www.kbase101.com/question/23589.html 这里介绍了acpi_osi的作用

安装 CUDA 也可以加参数，和安装驱动差不多，如果显示卡不是 N 卡，建议加上 --no-opengl-libs 参数，否则会覆盖非 N 卡的 OpenGL，导致无法显示。如果显示卡是 N 卡，则 X 的配置文件，/etc/X11/xorg.conf 可能需要修改。某些情况，nvidia-xconfig 能被用来自动生成系统文件 xorg.conf 文件，因此加上 --run-nvidia-xconfig 参数可以配置 X。
